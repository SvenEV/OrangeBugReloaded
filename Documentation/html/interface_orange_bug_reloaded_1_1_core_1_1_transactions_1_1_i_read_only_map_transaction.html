Detailed Description

The base interface for transactions.

Transactions are objects that simulate certain actions on a map and record changes that would be made to the map. Eventually if the transaction is not cancelled these changes are committed, i.e. applied to the underlying map, in an atomic way.

Transactions can be chained together. This way a transaction can override changes of earlier transactions.
